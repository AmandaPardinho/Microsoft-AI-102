USAR OS SERVIÇOS DE IA DO AZURE PARA APLICATIVOS EMPRESARIAIS

-gerenciamento, altenticação, segurança de rede para serviços de inteligência atificial do azure
    -segurança é de responsabilidade de quem implementa
    -cloud provider => oferece estratégias, ferramentas e recursos para tudo aquilo que for criado na nuvem
    -regenerar as chaves regularmente para proteger o acesso
        -proteção das chaves no Azure key vault => os aplicativos podem usar uma entidade de serviço como uma identidade gerenciada para recuperar chaves do key vault (cofre de chaves)
        -impede a exposição de chaves de acesso para usuários não autorizados
-gerenciamento de recursos de IA
-gerenciamento de custos e exibição de métricas
    -métricas são valores numéricos
    -métricas são coletadas em intervalos regulares
    -métricas são armazenadas em um banco de dados de série temporal
-gerenciamento de logs e alertas
    -o azure permite a criação de alertas para monitorar o uso de recursos
    -permite que a equipe correta saiba quando surgir um problema
    -os alertas se baseiam em uma regra pré programada (métrica)
    -configurações de diagnóstico fornecem informações detalhadas para diagnóstico e auditorias
    -destinos de diagnóstico =>
        -worklspace do log analytics
        -hubs de eventos
        -armazenamento do azure
    -os logs contêm informções com timestamp sobre as alterações feitas nos recursos
    -os dados dos logs são organizados em registro
-imagens de contêiners estão disponíveis para APIs de serviços de IA do azure usadas com frequência
    -implantação de contêineres =>
        -hosts locais do docker
        -instâncias de contêiner do azure
        -clusters dos serviços de kubernetes do azure
